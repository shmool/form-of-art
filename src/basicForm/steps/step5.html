We add error messages using ngMessages.

  • ngMessages is not a part of the ng module. You must add this module explicitly (npm install angular-messages and add it as a dependancy to your module).
  • ngMessages receives as an attribute the relevant $error object. (Here we pass the form's $error object, but you can pass the input's $error object to show the errors of the specific input.)
  • ngMessage, nested within ngMessages, holds the error message for the specific type(s) of error.
  • We use ngMessagesInclude (also nested within ngMessage) which allows us to hold our error messages in a separate file and reuse it.
  • A little more about ngMessages in step 10

ngMessages docs: https://docs.angularjs.org/api/ngMessages

Notice that the error messages appear when the form is loaded and as the user types. This is not very user-friendly. We must decide when the error messages should and should not appear. Also, this means we have a digest cycle on every change in the form. We would like to eliminate unnecessary digest cycles.
